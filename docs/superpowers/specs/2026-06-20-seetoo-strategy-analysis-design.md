# seetoo-strategy-analysis 設計規格

- 日期：2026-06-20
- 狀態：設計（依 /goal 自主推進，產出後供使用者複審）
- 依賴：`seetoo-strategy-core`（必須先存在；本技能站在其上）

---

## 0. 範圍

三支技能的第二支（建造順序：核心 → **分析** → 拷問 → 學習）。
本 spec 只涵蓋 `seetoo-strategy-analysis`（策略分析引擎）。

## 1. 定位

- **輸入**：一個事業／公司的描述（使用者提供的事實，或一個個案）。
- **輸出**：一份**有紀律、扣回司徒達賢方法**的結構化策略分析報告。
- **角色**：把 `seetoo-strategy-core` 的框架「拿來用」在某個具體事業上。它**不重述**講義內容，
  而是**援引**核心、把方法**施加**到輸入的事業。

## 2. 兩種「不杜撰」風險（本技能的最高紀律，對應使用者鐵則）

分析引擎同時面對兩種捏造風險，必須分流處理：

| 風險 | 防線 |
|---|---|
| (A) 誤述司徒達賢的方法/術語 | 一律**錨回核心**（引用 core 的 reference＋章節出處）；鎖術語；守反主張邊界 |
| (B) **捏造被分析事業的事實** | **只用使用者提供的事實**；任何推論一律標〔推測〕/〔假設〕/〔待問〕，不得寫成既成事實 |

報告中嚴格分區：**【事實】**（使用者給的）／**【框架】**（來自 core，附出處）／**【判斷】**（分析師推論，標推測）。

## 3. 行為：分析程序

以核心 `references/05-十步驟程序.md` 為主程序骨幹，套到輸入事業：

1. **描述現狀**（用 intake 清單蒐集；缺的標〔待問〕）
2. **現行策略的策略形態**：六大構面（產市垂規地競）逐面描述
3. **現行策略構想**：六大構面間如何配合、與環條目（環境/條件/目標組合）如何呼應 → 列出**前提**
4. **從環條目檢討**現行策略適用性
5. **構思策略選項**（六大構面排列組合，至少 2–3 個方案，含維持現狀）
6. **找出並驗證前提**（各方案的關鍵前提、真實性評估）
7. **建議未來策略形態**＋理由
8. **功能政策與組織調整**重點
9.（選配）**策略矩陣速寫**：價值鏈價值單元 × 六大構面的文字矩陣
10.（選配）行動計畫骨架

**兩種模式**：
- **完整分析**（預設）：步驟 1–8（±9/10）。
- **形態速寫**（輸入資訊少時）：步驟 1–3 + 關鍵前提。

> 反主張紀律：**不**從 SWOT / mission-vision / 環境分析 / 設定目標 起手（守 core `09`）；
> 先有具體策略形態與方案，才深入驗證環條目（司徒達賢主張）。

## 4. 檔案結構

```
skills/seetoo-strategy-analysis/
├── SKILL.md                       # 觸發、分析程序、紀律、消費 core、輸出結構、兩模式
└── references/
    ├── intake-清單.md              # 描述現狀要蒐集什麼（六大構面+環條目所需事實）
    ├── 分析報告範本.md             # 輸出報告的固定區段（含【事實】/【框架】/【判斷】分區）
    └── worked-example-鬍鬚張.md     # 以核心案例庫的鬍鬚張完整走一遍，示範品質與分區
```

## 5. 消費核心（穩定檔名 API）

| 階段 | 讀 core 的 |
|---|---|
| 全程術語 | `00-術語表.md` |
| 主程序 | `05-十步驟程序.md` |
| 六大構面/環條目 | `01-觀念架構與六大構面.md` |
| 策略構想/前提 | `02-策略構想與前提驗證.md` |
| 構思/評估方案 | `03-策略方案-構思評估選擇.md` |
| 功能政策/組織 | `04-功能政策與組織.md` |
| 策略矩陣（選配） | `06-策略矩陣與產業矩陣.md` |
| 道理透鏡（驗證前提時） | `07`、`08` |
| 起手式紅線 | `09-反主張與方法比較.md` |

消費方式：SKILL.md 指示「先讀／載入上列 core 檔，再依程序施作」（core 已 symlink 在 `~/.claude/skills/`）。

## 6. 驗證

- **worked example**：對鬍鬚張（用 core `10-案例庫.md` 的事實）跑完整分析，產物須：六大構面六全名、
  策略構想含前提、至少 2 方案、【事實/框架/判斷】分區清楚、框架處附 core 出處。
- **紀律測試**：(a) 餵一個資訊極少的事業 → 須走形態速寫並大量標〔待問〕，不得編事實；
  (b) 確認輸出未從 SWOT/目標 起手；(c) 術語鎖未踩線。
- **核對 example**：worked example 引用到的 core 內容，須與 core 檔一致（抽查）。

## 7. 非目標（YAGNI）

- 不是拷問教練（不反問使用者逼思考）、不是學習助教（不教學/出題）。
- 不捏造被分析事業的事實；不做一般 MBA 式 SWOT/五力起手分析。
- 不重抄講義內容（援引 core 即可）。
- 產業矩陣完整建模不在此（策略矩陣僅速寫）。
