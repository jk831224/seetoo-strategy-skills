# seetoo-strategy-tutor 設計規格

- 日期：2026-06-20
- 狀態：設計（依 /goal 自主推進，產出後供使用者複審）
- 依賴：`seetoo-strategy-core`

---

## 0. 範圍

三支技能的第四支、最後一支（core → analysis → grilling → **tutor**）。本 spec 只涵蓋
`seetoo-strategy-tutor`（講義學習助教）。

## 1. 定位

- **輸入**：使用者想學的主題／想被測驗／提交要批改的答案。
- **輸出**：**對教材的掌握**（學會司徒達賢的方法）。
- **角色**：帶使用者**讀懂並內化**這套講義——講解、帶練、出題、批改，全部扣回原文。

## 2. 與其他技能的區別

| 技能 | 做什麼 |
|---|---|
| tutor（本支） | **教**使用者學會方法（入門/內化） |
| analysis | 用方法**替使用者產出**分析報告 |
| grilling | **拷問**使用者既有思路、逼出嚴謹 |

tutor 是「學會」，語氣**鼓勵、引導**（與 grilling 的逼問不同）。

## 3. 四種模式

1. **講解**：解釋一個概念（六大構面/策略構想/前提/矩陣/道理/反主張），附 core 原文與出處；用類比與白話幫助理解（類比標為教學鷹架）。
2. **帶練**：帶使用者一步步把概念套到一個例子（用 core `10-案例庫` 的鬍鬚張/A/B，或使用者自己的例子）。
3. **出題**：依 `references/出題與批改規準.md` 出概念題／應用題／案例題。
4. **批改**：對照方法評分——有沒有用六大構面？有沒有找出前提、分辨事實vs待驗證？有沒有誤從 SWOT 起手？術語對不對？給具體回饋與訂正方向。

## 4. 紀律

1. **忠實教學**：只教講義內容；講解附 core 原文＋章節出處；鎖術語。
2. **不杜撰**：超出講義的，明說「講義未涵蓋」，不自行延伸成「司徒達賢說」。
3. **兩層標示**：**【講義】**（引自 core，附出處）／**【教學鷹架】**（tutor 自加的類比、練習、提示——明標、不冒充原文）。
4. **鼓勵式**：引導、給線索、肯定進步；不像 grilling 那樣逼問（需要被拷問時，引導使用者去用 grilling 技能）。

## 5. 檔案結構

```
skills/seetoo-strategy-tutor/
├── SKILL.md                       # 四模式、教學紀律、消費 core、學習路徑入口、自檢
└── references/
    ├── 學習路徑.md                 # 建議學習順序＋各階段學習目標（扣 core 各檔）
    ├── 出題與批改規準.md           # 題型範例＋評分規準（對照方法的檢核點）
    └── worked-example-教學.md       # 一段示範教學（講解→帶練→出題→批改一例）
```

## 6. 學習路徑（概念順序，扣 core）

1. 為什麼是「思維方式」、策略形態總覽〔core `01`〕
2. 六大構面：產市垂規地競，逐面〔core `01`〕
3. 策略構想與前提（既存事實 vs 待驗證）〔core `02`〕
4. 產生與評估策略方案〔core `03`〕
5. 功能政策與組織如何配合〔core `04`〕
6. 完整十步驟程序〔core `05`〕
7. 策略矩陣與產業矩陣〔core `06`〕
8. 道理（經濟／組織決策）當透鏡〔core `07`/`08`〕
9. 為何不從 SWOT/目標 起手（方法的邊界）〔core `09`〕
10. 用案例庫貫穿複習〔core `10`〕

## 7. 出題與批改

- **題型**：概念題（定義/區辨，如「環境前提 vs 條件前提」）、應用題（給情境，套六大構面/找前提）、案例題（用案例庫）。
- **評分規準（檢核點）**：是否用對術語與六大構面、是否找出並分類前提、是否分辨事實vs待驗證、是否避免 SWOT/目標 起手、因果是否講清楚。回饋須具體並指回 core 出處。

## 8. 消費 core（穩定檔名 API）

全部 11 份皆可用；依模式/主題讀對應檔（學習路徑第 N 階段 ↔ core 第 N 類檔），術語以 `00-術語表` 為準。

## 9. 驗證

- **worked example 教學**：完整示範「講解策略構想 → 帶練（鬍鬚張）→ 出一題 → 批改一份示範答案」，全程【講義】/【教學鷹架】分層、附 core 出處。
- **紀律測試**：(a) 問講義沒有的（如「司徒達賢怎麼看 OKR」）→ 必須答「講義未涵蓋」；(b) 批改時對照方法檢核點、不放水也不杜撰；(c) 術語鎖未踩線；(d) 鼓勵式語氣（與 grilling 區隔）。

## 10. 非目標（YAGNI）

- 不替使用者做完整策略分析（那是 analysis）、不逼問拷問（那是 grilling）。
- 不杜撰講義以外內容當「司徒達賢說」。
- `sources/` 的例題/習題簡報（`.ppsx`）與產業矩陣 `.xls` 暫不解析納入（未來增強；本版以 core 既有內容出題）。
