# 出題與批改規準

## 題型（三類）

1. **概念題**：定義與區辨。例：
   - 用一句話說明「策略構想」是什麼？〔core `02`〕
   - 區辨「環境前提」與「條件前提」，各舉一例。〔core `02`/`10`〕
   - 六大構面是哪六個？（要全名）〔core `01`〕
2. **應用題**：給一個簡短情境，要求套方法。例：
   - 給定一家「主打現做、價格偏高的便當店」，用六大構面描述其策略形態，並列出 2 個關鍵待驗證前提。
3. **案例題**：用 core `10-案例庫`。例：
   - 鬍鬚張 1979 要不要開店面？列出該方案的環境前提與條件前提，以及「前提不成立」的後果。〔core `10` §D〕

> 出題紀律：題目與標準答案都要能在 core 找到依據；標準答案附 core 出處。
> 不出「講義未涵蓋」的題（不考 OKR、五力細節等非本講義主張的東西）。

## 批改檢核點（評分對照方法，非對照感覺）

逐項打勾並給具體回饋（指回 core）：

- [ ] **術語正確**：用講義術語、六大構面用全名（不寫成 positioning/定位/STP）。
- [ ] **六大構面**：該用形態描述時，有沒有逐面落地、而非空泛形容？
- [ ] **找出前提**：有沒有把策略背後的前提講出來？
- [ ] **事實 vs 待驗證**：有沒有分辨哪些是既存事實、哪些是待驗證假設？（最常見扣分點）
- [ ] **不從 SWOT/目標 起手**：有沒有誤把 SWOT/設定目標 當分析起點？（踩到要指出並導正，core `09`）
- [ ] **因果**：策略構想的因果/配合關係有沒有講清楚、站得住？

## 回饋格式（鼓勵式）
1. 先肯定對的地方（具體指出哪裡用對了方法）。
2. 指出 1–2 個最關鍵的缺漏，附 core 出處與一句訂正方向。
3. 給一個「再想一下」的小提示（**不直接給答案**，引導他自己補上）。
4. 需要的話，標【教學鷹架】補一個類比或口訣。

> 批改不放水也不杜撰：找不到依據的不要亂給分；標準答案以 core 為準。
