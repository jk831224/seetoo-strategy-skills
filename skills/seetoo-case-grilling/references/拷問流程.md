# 拷問流程

一場拷問的節奏。每一步以**問**推進；對方含糊就停在原地追問，不前進。

## 0. 開場
- 請對方用幾句話講：這是什麼事業／你的策略主張是什麼／想被拷問的決定是什麼。
- 不要先評論。聽完先丟第一個「事實 vs 推測」的問題（提問庫 §1）。

## 1. 形態探問（站穩現況）　〔core `01`〕
- 用六大構面（產市垂規地競）逐面逼具體做法。哪一面講不清楚，就卡在那面追問。
- 目標：把「品質好、很厲害」這類抽象賣點，逼成可被檢驗的具體做法。

## 2. 構想／前提探問（核心戰場）　〔core `02`〕
- 逼出策略構想：六大構面之間、與環條目之間如何配合呼應。
- 對每個「配合」追問：這是**既存事實**還是**待驗證前提**？怎麼驗證？
- 用「前提不成立會怎樣」逼出具體後果。

## 3. 因果壓力測試　〔core `10`/`07`/`08`〕
- 把對方的「A→B」攤開，逐步檢查每一環；找最脆弱的因果與會破壞它的力量。

## 4. 反主張攔截（隨時）　〔core `09`〕
- 一旦對方滑向 SWOT / mission / 環境分析 / 設定目標 起手，立刻點出並導回策略形態。

## 5. 收斂與摘要
- 收的時機：關鍵前提講清楚並能分辨事實 vs 待驗證；或盲點充分暴露；或對方喊停。
- 收尾固定給：**一句話策略形態摘要 ＋ 待驗證前提清單 ＋ 下一步要驗證什麼**。

## 紀律提醒
- 全程**以問為主**；發現自己開始「替他分析、下結論」就收手，改成把那個結論變成一個問題丟回去。
- **不腦補**對方公司的任何事實——不知道就問。
- 一次別丟太多問題；逼他**自己**想清楚一個，再前進。
