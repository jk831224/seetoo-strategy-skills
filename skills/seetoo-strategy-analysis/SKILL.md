# seetoo-strategy-analysis（策略分析引擎）

把司徒達賢的策略形態分析法，**施加到一個具體事業**，產出結構化、可回溯的策略分析報告。
本技能**不重述**講義內容——它**援引** `seetoo-strategy-core` 並把方法用在輸入的事業上。

## 先載入核心

動工前先讀核心（已 symlink 在 `~/.claude/skills/seetoo-strategy-core/`）：
- `references/05-十步驟程序.md`（主程序骨幹）
- `references/00-術語表.md`（術語鎖）
- `references/09-反主張與方法比較.md`（起手紅線）

其餘依步驟需要再讀（見下方對照表）。

## 最高紀律：兩種「不杜撰」

| 風險 | 防線 |
|---|---|
| 誤述司徒達賢的方法/術語 | 框架一律**錨回 core**（附 core 檔名＋章節出處）；**鎖術語**不換成一般 MBA 詞 |
| **捏造被分析事業的事實** | **只用使用者提供的事實**；缺的標〔待問〕；推論標〔推測〕/〔假設〕，**不得寫成既成事實** |

報告全程**三分區**：
- **【事實】**——使用者提供的事業事實。
- **【框架】**——來自 core 的方法/概念（附 core 出處）。
- **【判斷】**——分析師的推論與建議（標〔推測〕）。

**起手紅線**：**不**從 SWOT / mission-vision / 環境分析 / 設定目標 開始（守 core `09`）。
依司徒達賢主張：**先有具體策略形態與方案，才深入驗證環條目**。

## 程序（依 core `05` 十步驟）

1. **描述現狀**——用 `references/intake-清單.md` 蒐集事實；缺漏標〔待問〕。
2. **現行策略形態**——六大構面（**產市垂規地競**）逐面描述。〔框架→core `01`〕
3. **現行策略構想**——六大構面間如何配合、與環條目（環境/條件/目標組合）如何呼應；**列出前提**。〔core `02`〕
4. **從環條目檢討**現行策略的適用性。〔core `01`/`02`〕
5. **構思策略選項**——六大構面排列組合，至少 2–3 個方案（含維持現狀）。〔core `03`〕
6. **找出並驗證前提**——各方案關鍵前提＋真實性評估；必要時用道理當透鏡。〔core `02`/`07`/`08`〕
7. **建議未來策略形態**＋理由。〔core `03`〕
8. **功能政策與組織調整**重點。〔core `04`〕
9.（選配）**策略矩陣速寫**——價值單元 × 六大構面的文字矩陣。〔core `06`〕
10.（選配）**行動計畫**骨架。

## 兩種模式

- **完整分析**（預設）：步驟 1–8（可加 9/10）。
- **形態速寫**（輸入資訊少時）：步驟 1–3 ＋ 關鍵前提；大量標〔待問〕，**不要靠想像補事實**。
